### 2. 后端已完成

| 模块 | 文件 | 实现内容 |
|------|------|----------|
| 应用入口 | `backend/main.py` | FastAPI 实例、CORS 中间件、路由挂载、启动时初始化数据库、音频静态文件挂载（`/api/audio`） |
| 配置管理 | `backend/config.py` | Pydantic Settings，支持 .env 文件，自动创建数据目录 |
| 数据库 | `backend/database.py` | SQLite 建表（history、settings）、异步读写、设置项存取 |
| 数据模型 | `backend/models.py` | 6个请求模型：SynthesizeRequest、VoiceDesignRequest、VoiceCloneRequest、ValidateKeyRequest、HistoryRecord、SettingsUpdate |
| 合成路由 | `backend/routers/synthesize.py` | POST /api/synthesize、/api/voice-design、/api/voice-clone、/api/validate-key；自动保存音频文件、写入历史记录 |
| 音色路由 | `backend/routers/voices.py` | GET /api/voices（返回8种预置音色）、/api/voice-preview/{id}（占位） |
| 历史路由 | `backend/routers/history.py` | GET /api/history（分页+搜索+筛选）、GET/DELETE /api/history/{id}、DELETE /api/history（清空）、PUT 收藏切换 |
| 设置路由 | `backend/routers/settings.py` | GET/PUT /api/settings |
| API客户端 | `backend/services/tts_client.py` | 封装三种模型的调用方法、消息构造逻辑、风格标签注入、Key验证 |
| 音频处理 | `backend/services/audio_processor.py` | PCM16→WAV 格式转换（添加44字节文件头）、Base64 解码 |

## 二、后续计划与步骤

### 第一阶段：核心功能补全（优先级 P0）

#### 1. 音频文件静态服务
- **目标**：历史记录中的音频文件可以通过 URL 直接访问
- **步骤**：在 `backend/main.py` 中添加 `StaticFiles` 挂载，将 `data/audio/` 目录映射到 `/api/audio` 路径
- **验证**：前端播放器能通过 `/api/audio/2026/04/xxx.wav` 播放音频

#### 2. 音色试听功能
- **目标**：8种预置音色可以试听
- **步骤**：
  - 准备试听音频文件（统一文本"欢迎使用MiMo语音合成"）
  - 将音频文件放入 `data/preview/` 目录
  - 实现 `GET /api/voice-preview/{voice_id}` 接口返回对应音频
  - 前端音色选择器增加试听按钮
- **验证**：点击试听按钮可播放对应音色的示例音频

#### 3. API Key 未配置引导
- **目标**：首次使用时引导用户配置 API Key
- **步骤**：
  - 后端合成接口增加 Key 检查，未配置时返回 401
  - 前端全局拦截 401 响应，弹出引导弹窗
  - 弹窗中嵌入 API Key 输入框和验证按钮
- **验证**：未配置 Key 时点击合成，弹出配置引导

#### 4. 错误处理完善
- **目标**：统一错误提示，覆盖常见异常场景
- **步骤**：
  - 后端统一异常处理中间件（网络超时、API 返回错误、文件过大等）
  - 前端 axios 拦截器统一处理错误消息
  - 合成失败时保留用户输入内容
- **验证**：断网、Key 无效、文本为空等场景均有友好提示

### 第二阶段：体验优化（优先级 P1）

#### 5. 音频波形显示
- **目标**：AudioPlayer 组件显示音频波形图
- **步骤**：
  - 使用 Web Audio API 的 `AnalyserNode` 获取频域数据
  - 使用 Canvas 绘制波形
  - 播放时显示实时频谱
- **验证**：播放音频时显示动态波形

#### 6. 深色主题
- **目标**：支持浅色/深色/跟随系统三种主题
- **步骤**：
  - CSS 变量定义主题色
  - Element Plus 暗色模式切换
  - 设置页面切换后实时生效
- **验证**：切换深色主题后界面颜色正确

#### 7. 文本清洗增强
- **目标**：一键清洗文本中的无关字符
- **步骤**：
  - 去除 HTML 标签
  - 去除多余空白和换行
  - 去除特殊控制字符
  - 可选：数字/日期中文读法转换
- **验证**：粘贴含 HTML 的文本，点击清洗后变为纯文本

#### 8. 快捷键支持
- **目标**：常用操作支持键盘快捷键
- **步骤**：
  - `Ctrl+Enter`：开始合成
  - `Space`：播放/暂停
  - `Ctrl+S`：下载音频
- **验证**：按下快捷键触发对应操作

### 第三阶段：功能扩展（优先级 P2）

#### 9. 批量合成
- **目标**：支持多段文本逐条合成并打包下载
- **步骤**：
  - 前端添加批量输入区（多行文本，每行一条）
  - 后端新增 `POST /api/batch-synthesize` 接口
  - 逐条调用 API，返回进度
  - 完成后打包为 ZIP 下载
- **验证**：输入多行文本，逐条合成并下载 ZIP

#### 10. 历史记录批量导出
- **目标**：选中多条记录打包导出
- **步骤**：
  - 前端表格增加多选框
  - 后端新增 `POST /api/history/export` 接口
  - 将选中记录的音频文件打包为 ZIP
- **验证**：选中记录后点击导出，下载 ZIP 文件

#### 11. 数据导入/导出
- **目标**：支持历史记录和设置的 JSON 导入导出
- **步骤**：
  - `GET /api/export` 导出所有数据为 JSON
  - `POST /api/import` 从 JSON 导入
  - 前端设置页面添加导入导出按钮
- **验证**：导出后删除数据，再导入恢复

#### 12. 存储空间管理
- **目标**：查看磁盘占用，清理旧记录
- **步骤**：
  - 后端 `GET /api/storage` 返回音频文件总量和记录数
  - `DELETE /api/history/older-than/{days}` 删除指定天数前的记录
  - 前端设置页面展示存储概览和清理操作
- **验证**：查看存储占用，清理 30 天前的记录

### 第四阶段：打包与发布（优先级 P3）

#### 13. 前端构建集成
- **目标**：后端启动时自动加载前端构建产物

- **步骤**：

  - 处理各平台兼容性问题
- **验证**：各平台安装后功能正常

#### 16. 文档与发布
- **目标**：完善项目文档，发布 GitHub Release

## 三、技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 后端框架 | FastAPI | 异步支持好，自动生成 API 文档，Pydantic 集成 |
| 前端框架 | Vue 3 + Element Plus | 组件丰富，中文生态好，上手快 |
| 桌面方案 | PyWebView | 轻量，Python 原生，无需 Electron 体积 |
| 数据库 | SQLite | 零配置，单文件，适合桌面应用 |
| 音频存储 | 本地文件系统 | 简单直接，音频文件按日期分目录 |
| API 调用 | httpx | 异步支持，与 FastAPI 配合好 |
| 状态管理 | Pinia | Vue 3 官方推荐，轻量 |
| 构建工具 | Vite | 快速，配置简单，Vue 官方默认 |

## 四、已知待解决问题

1. **API 接口细节未验证**：MiMo API 的实际请求/响应格式需根据真实 API 文档调整
2. **流式输出**：当前为兼容模式，待官方升级后适配
3. **音色克隆会话管理**：API 限制同一会话不可更换音频样本，需在前端给出明确提示
4. **音频时长估算**：当前基于字符数粗略估算，实际时长受语速、停顿影响
5. **WebView 兼容性**：不同操作系统的 WebView 引擎可能有渲染差异
