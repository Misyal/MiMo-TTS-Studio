# 📋 MiMo-V2.5-TTS 桌面语音合成工具 — 完整计划书

## 一、项目概述

### 1.1 项目名称

**MiMo TTS Studio**

### 1.2 项目定位

基于小米 MiMo-V2.5-TTS 开放 API，开发一款**开箱即用的桌面语音合成工具**。用户下载安装后，双击即可运行，通过图形界面完成所有语音合成操作，无需任何命令行或编程知识。

### 1.3 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| 短视频创作者 | 快速生成配音音频，支持多种情绪和方言 |
| 有声书制作者 | 使用温柔音色朗读长文本，支持章节批量合成 |
| 播客主持人 | 生成片头片尾语音、广告口播 |
| 音乐爱好者 | 输入歌词生成演唱音频 |
| 普通用户体验 | 尝试不同音色、克隆自己的声音 |

### 1.4 核心卖点

1. **零门槛使用**：下载安装即用，无需配置 Python 环境或命令行
2. **三种合成模式**：预置音色 / 文本描述定制音色 / 音频样本克隆音色
3. **丰富的风格控制**：支持情绪、语调、方言、语速等自然语言控制
4. **唱歌能力**：输入歌词即可生成演唱音频
5. **本地运行**：界面和历史记录存储在本地，仅合成时联网

## 二、官方 API 核心信息整理

### 2.1 服务接入信息

- **服务地址**：`https://api.xiaomimimo.com/v1`
- **认证方式**：每个请求的头部中携带 `api-key` 字段，值为用户申请的 API Key
- **协议格式**：完全兼容 OpenAI Chat Completions API 格式（使用标准的 HTTP POST 请求，请求体为 JSON 格式）
- **音频输出参数**：PCM16LE 编码、单声道、24000Hz 采样率

### 2.2 三种模型详细说明

#### 模型一：MiMo-V2.5-TTS（预置音色）

- **模型标识**：`mimo-v2.5-tts`
- **功能描述**：使用官方提供的 8 种精品音色进行语音合成
- **音色指定方式**：在请求中通过 `voice` 字段指定音色 ID（如 `bingtang`）
- **支持的音色**：冰糖、茉莉、苏打、白桦、Mia、Chloe、Milo、Dean
- **特殊能力**：支持唱歌模式——在待合成文本的最前面添加"（唱歌）"标记即可触发
- **唱歌限制**：仅此模型支持唱歌，歌词建议使用中文，英文歌词效果可能不佳

#### 模型二：MiMo-V2.5-TTS-VoiceDesign（文本描述定制音色）

- **模型标识**：`mimo-v2.5-tts-voicedesign`
- **功能描述**：通过自然语言描述目标音色特征，系统根据描述生成全新音色，无需提供任何音频样本
- **音色指定方式**：在请求的用户消息中，用自然语言描述目标音色（如"年轻女性，温柔治愈系，语速缓慢"）
- **必须要求**：用户消息中必须包含音色描述内容，否则请求会失败
- **描述建议**：可包含性别、年龄、音色特点、语速、情绪等维度，描述越具体效果越好

#### 模型三：MiMo-V2.5-TTS-VoiceClone（音频样本音色克隆）

- **模型标识**：`mimo-v2.5-tts-voiceclone`
- **功能描述**：基于用户提供的音频样本，复刻该声音来说出任意文本
- **音色指定方式**：在请求中附带 Base64 编码的音频样本数据
- **音频格式要求**：仅支持 mp3 和 wav 两种格式
- **大小限制**：音频文件 Base64 编码后的总大小不能超过 10MB
- **会话限制**：同一会话内音频样本不可更改，如需使用不同样本需创建新会话

### 2.3 预置音色完整清单

| 音色名称 | 音色 ID | 语言 | 性别 | 适合场景 |
|----------|---------|------|------|----------|
| 冰糖 | `bingtang` | 中文 | 女性 | 通用场景，发音清晰自然 |
| 茉莉 | `moli` | 中文 | 女性 | 温柔亲切，适合有声书、睡前故事 |
| 苏打 | `suda` | 中文 | 男性 | 年轻活力，适合短视频、游戏解说 |
| 白桦 | `baihua` | 中文 | 男性 | 沉稳大气，适合新闻播报、纪录片 |
| Mia | `Mia` | 英文 | 女性 | 美式英语，通用场景 |
| Chloe | `Chloe` | 英文 | 女性 | 美式英语，温柔知性 |
| Milo | `Milo` | 英文 | 男性 | 美式英语，年轻阳光 |
| Dean | `Dean` | 英文 | 男性 | 美式英语，成熟稳重 |

### 2.4 风格控制机制详解

#### 机制一：自然语言控制

- **放置位置**：请求中的用户消息部分
- **使用方式**：用自然语言描述期望的说话风格，系统会自动解析并应用
- **示例描述**：
  - "用温柔的语气慢慢说"
  - "语速快一些，带点兴奋的感觉"
  - "模仿东北方言来说"
  - "像新闻主播一样正式播报"
  - "带一点委屈的哭腔"

#### 机制二：音频标签控制

- **放置位置**：请求中的助手消息里，待合成文本的最前面
- **使用方式**：在文本开头添加用括号包裹的风格标签
- **支持的标签分类**：

| 分类 | 可用标签 |
|------|----------|
| 情绪 | 高兴、悲伤、愤怒、恐惧、厌恶、惊讶、困惑 |
| 哭泣 | 呜咽、啜泣、抽泣、嚎哭 |
| 呼吸 | 喘息、叹气 |
| 语调 | 低语、柔和、兴奋、沮丧、害羞、强调、恐惧（紧张颤抖）、快语（急促含糊） |
| 语速 | 慢速、快速 |
| 方言 | 东北话、四川话、河南话、陕西话、粤语 |
| 特殊 | 唱歌（仅限 mimo-v2.5-tts 模型） |

#### 组合使用规则

- 两种机制可以同时使用，互不冲突
- 自然语言控制放在用户消息中，音频标签放在助手消息的文本开头
- 多个音频标签可以组合使用，如"（东北话，兴奋）哎呀妈呀太好了！"

### 2.5 请求结构规则

一个完整的语音合成请求由以下部分组成：

1. **模型字段**：指定使用哪种模型（三选一）
2. **消息数组**：包含以下顺序的消息
   - 系统消息（可选）：设置整体场景规则，如"你是一个温柔的女性助手"
   - 用户消息（按场景要求）：
     - 预置音色模式：可选，用于传递风格指令
     - 音色设计模式：必须，用于传递音色描述
     - 音色克隆模式：可选，用于传递风格指令
   - 助手消息（必须）：包含待合成的目标文本，可选添加音频标签前缀
3. **音频格式字段**：指定输出格式为 `wav` 或 `pcm16`
4. **流式字段**：设为 `true` 时为流式模式（当前为兼容模式）

### 2.6 响应格式

- **非流式响应**：返回一个完整的 JSON 对象，其中包含 Base64 编码的音频数据
- **流式响应**：返回多个数据块，每个块包含一小段 Base64 编码的音频数据，需按顺序拼接（当前为兼容模式，实际是推理完成后一次性返回）
- **解码方式**：将 Base64 字符串解码为二进制数据，即为原始音频数据
- **格式转换**：如果是 pcm16 格式，需添加 WAV 文件头后才能直接播放

### 2.7 当前限制与注意事项

1. **流式输出**：当前为兼容模式，不是真正的逐帧流式（推理完成后一次性返回），官方后续会升级为真正流式
2. **流式格式**：流式调用时音频格式必须选择 `pcm16`
3. **克隆大小**：音色克隆的音频样本 Base64 编码后不能超过 10MB
4. **唱歌限制**：唱歌模式仅支持 `mimo-v2.5-tts` 模型，歌词建议使用中文
5. **计费策略**：当前限时免费，后续可能调整
6. **描述匹配**：音色设计模式下，音色描述与合成文本需逻辑匹配（如描述为女性音色时，合成文本应适合该音色）

## 三、产品功能详细设计

### 3.1 整体界面结构

采用**左侧固定导航栏 + 右侧动态内容区**的经典桌面应用布局。

#### 左侧导航栏

- **顶部**：应用 Logo（MiMo TTS Studio）和版本号
- **中部**：五个功能模块的导航入口，从上到下依次为：
  1. 🎙️ 文本合成
  2. 🎨 音色设计
  3. 🔄 音色克隆
  4. 📋 历史记录
  5. ⚙️ 设置
- **底部**：当前连接状态指示灯（绿色=正常，红色=异常）和 API Key 状态文字

#### 右侧内容区

- **顶部**：当前模块标题和一句话功能说明
- **中部**：该模块的具体操作界面
- **底部**：状态栏（显示当前操作状态、错误提示等）

### 3.2 模块一：文本合成

#### 功能目标

使用 8 种预置音色进行语音合成，支持风格控制和唱歌模式。

#### 界面元素

**左侧面板（输入区）**：

1. **文本输入框**
   - 多行文本输入区域，支持滚动
   - 右上角显示当前字符数和最大字符数（建议限制 2000 字）
   - 底部显示预估合成时长（基于中文约 4 字/秒、英文约 3 词/秒估算）
   - 支持从剪贴板粘贴文本

2. **音色选择区**
   - 下拉选择框，展示所有 8 种预置音色
   - 每个选项显示格式：音色名称 + 语言标签 + 性别标签（如"冰糖 | 中文 | 女性"）
   - 选中后在下方显示该音色的详细描述和适合场景
   - 每个音色右侧提供"试听"按钮，点击播放该音色的示例音频（预置在应用内）

3. **风格控制区**
   - **快捷标签区**：展示常用风格的标签按钮，分为三行：
     - 第一行（情绪）：温柔、兴奋、悲伤、愤怒、低语
     - 第二行（方言）：东北话、四川话、河南话、陕西话、粤语
     - 第三行（语速）：慢速、快速
   - 点击标签按钮可选中/取消，支持多选，选中的标签高亮显示
   - **自定义输入框**：可输入任意自然语言风格描述，如"像老朋友聊天一样轻松"
   - **标签预览区**：实时展示将要注入到文本前的音频标签（如"（东北话，兴奋）"）

4. **唱歌模式开关**
   - 开关按钮，默认关闭
   - 开启后自动在文本前添加"（唱歌）"标记
   - 开启时如果选择的模型不支持唱歌，显示警告提示
   - 开启后显示提示文字："建议使用中文歌词以获得最佳效果"

5. **输出格式选择**
   - 两个单选按钮：wav（默认选中）和 pcm16
   - 选择 pcm16 时显示提示："流式调用时需使用此格式"

6. **合成按钮**
   - 大按钮，文字为"开始合成"
   - 点击后变为加载状态，显示进度动画和预估剩余时间
   - 合成过程中按钮不可重复点击
   - 合成失败时按钮恢复可用状态，并显示错误信息

**右侧面板（输出区）**：

1. **音频播放器**
   - 合成完成后自动出现
   - 包含播放/暂停按钮、进度条、音量控制、当前时间/总时长显示
   - 支持拖动进度条跳转播放位置
   - 支持循环播放开关

2. **波形显示**
   - 显示合成音频的波形图
   - 播放时有播放指针跟随

3. **操作按钮区**
   - "重新播放"按钮：从头播放
   - "下载"按钮：将音频保存到本地，弹出文件保存对话框
   - "收藏"按钮：将本次合成加入收藏（存储在历史记录中并标记）
   - "重新合成"按钮：清空当前结果，准备新的合成

## 四、技术实现方案

### 4.1 整体技术架构

应用采用**本地服务 + 内嵌浏览器窗口**的架构模式，具体分为三层：

#### 表现层（前端）

- **技术选型**：Vue 3 框架 + Element Plus UI 组件库
- **职责**：构建所有用户界面，处理用户交互，调用后端接口
- **运行环境**：嵌入在桌面窗口中的 WebView 引擎

#### 业务层（后端）

- **技术选型**：Python 3.9+ 语言 + FastAPI 框架
- **职责**：处理业务逻辑，调用 MiMo TTS API，管理音频数据，操作本地数据库
- **运行环境**：本地 HTTP 服务，仅监听本机地址

#### 桌面壳层

- **技术选型**：PyWebView 库
- **职责**：创建原生桌面窗口，嵌入前端页面，提供系统级功能（文件对话框、托盘图标等）
- **运行环境**：直接调用操作系统原生 API

### 4.2 各层详细设计

#### 桌面壳层设计

- 启动时创建一个固定尺寸的原生窗口（建议默认 1200×800 像素，支持调整大小）
- 窗口标题显示"MiMo TTS Studio"
- 窗口内嵌 WebView，加载 FastAPI 提供的前端页面
- 支持窗口最小化、最大化、关闭操作
- 关闭窗口时触发应用退出流程：停止本地服务、保存未保存的数据、释放资源
- 可选：支持系统托盘图标，最小化时隐藏到托盘

#### 后端服务设计

**服务启动流程**：
1. 应用启动时，自动在本地启动 FastAPI 服务
2. 监听地址为 `127.0.0.1`，端口随机选择一个未占用的端口（如 18700-18800 范围）
3. 启动成功后，将实际端口传递给桌面壳层，用于加载前端页面

**API 接口设计**（后端提供的本地接口）：

| 接口路径 | 方法 | 功能说明 |
|----------|------|----------|
| `/api/synthesize` | POST | 文本合成：接收文本、音色、风格等参数，调用 MiMo API，返回音频数据 |
| `/api/voice-design` | POST | 音色设计：接收音色描述和文本，调用 MiMo API，返回音频数据 |
| `/api/voice-clone` | POST | 音色克隆：接收音频文件（Base64）和文本，调用 MiMo API，返回音频数据 |
| `/api/voices` | GET | 获取预置音色列表：返回所有可用音色的元数据 |
| `/api/voice-preview/{voice_id}` | GET | 获取音色试听音频：返回指定音色的预置示例音频 |
| `/api/history` | GET | 获取历史记录列表：支持分页、筛选、搜索 |
| `/api/history/{id}` | GET | 获取单条历史记录详情 |
| `/api/history/{id}` | DELETE | 删除单条历史记录 |
| `/api/history` | DELETE | 清空所有历史记录 |
| `/api/settings` | GET | 获取当前设置 |
| `/api/settings` | PUT | 更新设置 |
| `/api/validate-key` | POST | 验证 API Key 有效性 |
| `/api/health` | GET | 健康检查：返回服务状态 |

**MiMo API 调用流程**：

1. 接收前端请求参数
2. 根据模型类型构造符合 MiMo API 格式的请求体
3. 使用 HTTP 客户端向 MiMo API 发送请求
4. 接收响应，提取 Base64 编码的音频数据
5. 将 pcm16 数据转换为 wav 格式（添加 WAV 文件头）
6. 将音频数据返回给前端，同时写入历史记录数据库

**音频格式转换逻辑**：

- 接收到 MiMo API 返回的 Base64 音频数据后，解码为二进制数据
- 如果输出格式为 pcm16：直接返回原始二进制数据
- 如果输出格式为 wav：在 pcm16 数据前添加标准的 WAV 文件头（44 字节），包含采样率 24000、单声道、16 位等参数信息
- 最终返回给前端的音频数据统一为 Base64 编码格式

#### 前端界面设计

**技术实现要点**：

- 使用 Vue 3 的组合式 API 构建组件
- 使用 Vue Router 实现页面路由（对应五个功能模块）
- 使用 Pinia 进行全局状态管理（当前设置、API Key、历史记录缓存等）
- 使用 Element Plus 提供的 UI 组件：按钮、输入框、下拉选择、开关、对话框、消息提示等
- 使用 Web Audio API 实现音频播放和波形显示
- 使用 axios 或 fetch 调用后端本地接口

**音频播放器实现**：

- 使用 Web Audio API 解码音频数据
- 使用 Canvas 绘制音频波形图
- 实现播放/暂停、进度拖拽、音量控制、循环播放功能
- 播放时在波形图上显示播放指针位置

**文件上传实现**（音色克隆模块）：

- 使用 HTML5 的拖拽 API 实现拖拽上传
- 使用 File API 读取文件信息（大小、类型）
- 使用 FileReader 将文件转为 Base64 编码
- 上传前校验文件格式和大小

### 4.3 数据存储设计

使用 SQLite 数据库存储以下数据：

#### 表一：历史记录表

存储字段包括：
- 记录唯一标识（自增主键）
- 合成时间（时间戳）
- 模型类型（文本合成 / 音色设计 / 音色克隆）
- 使用的音色信息（音色 ID 或音色描述或文件名）
- 风格标签内容
- 合成文本内容
- 音频数据（Base64 编码存储）
- 音频格式（wav / pcm16）
- 音频时长（秒）
- 是否收藏（布尔值）

#### 表二：设置表

存储字段包括：
- 设置项名称（主键）
- 设置项值

存储的设置项包括：API Key、默认音色、默认输出格式、默认保存路径、主题、语言等。

### 4.4 错误处理机制

#### API Key 相关错误

- **未配置 API Key**：弹出提示引导用户前往设置页面配置
- **API Key 无效**：返回明确错误信息"API Key 无效，请检查后重新输入"
- **API Key 过期或被禁用**：返回对应错误信息

#### 网络相关错误

- **网络不可用**：提示"网络连接不可用，请检查网络设置"
- **请求超时**：提示"请求超时，请稍后重试"，超时时间建议设为 30 秒
- **服务端错误（5xx）**：提示"服务暂时不可用，请稍后重试"

#### 音频相关错误

- **音频文件格式不支持**：提示"仅支持 mp3 和 wav 格式"
- **音频文件过大**：提示"文件大小超出限制，请选择更短的音频"
- **文本内容为空**：提示"请输入要合成的文本"
- **文本内容过长**：提示"文本内容超出字数限制"

#### 通用处理

- 所有错误均在界面底部状态栏或弹出消息框中显示
- 错误信息包含：错误类型、具体原因、建议操作
- 合成失败时不清除已输入的内容，方便用户修改后重试

## 七、风险评估与应对策略

| 风险项 | 发生概率 | 影响程度 | 应对策略 |
|--------|----------|----------|----------|
| MiMo API 服务变更或下线 | 中 | 高 | API 调用层做抽象封装，便于快速适配变更；持续关注官方更新动态；在应用中提供 API 状态检测功能 |
| 流式输出长期未上线 | 高 | 中 | 优化加载状态展示，提供进度动画和预估等待时间；在界面中说明当前为兼容模式；后续上线后第一时间适配 |
| 音频文件大小限制导致克隆失败 | 中 | 中 | 在上传时即时校验文件大小并给出明确提示；提供音频压缩建议；在文档中说明推荐的音频时长（建议 10-30 秒） |
